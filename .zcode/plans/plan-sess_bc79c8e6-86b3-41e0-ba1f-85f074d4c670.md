План миграции автообновления с WordPress/WP File Download на защищённый HTTP-сервис 1С.

## Контекст находок

- Облачная часть уже готова и возвращает ровно нужный JSON: `cloud/update_endpoint/index.py` преобразует `apkKey` из бакета в подписанную `apkUrl` (+`urlExpiresInSec`, `sha256`, `required`). Облако и 1С не трогаем.
- `DioClient` (lib/core/network/dio_client.dart) уже даёт Basic Auth + таймауты + failover по `remoteHosts` — переиспользуем для запроса манифеста к 1С.
- Существующий паттерн авторизованных репозиториев (`inventoryRepositoryProvider`): `Provider`, читающий `authControllerProvider.session!` и строящий `DioClient(BasicCredentials(session.login, session.password))`. Применю его же.
- `crypto` уже в pubspec.lock (транзитивно), добавлю как прямую зависимость.
- WordPress-привязка расползлась по: `app_config.dart`, `version_manifest.dart`, `update_repository.dart`, трём тестам и README. В `app_config.dart:62` лежит захардкоженный пароль WordPress `portalCredentials = ('services', '92!OrSqCt9oRJ*K!cwHF0^yd')` — удалить обязательно.

## Изменения

### 1. `lib/core/config/app_config.dart`
Удалить всё WordPress-связанное (используется только в update-функционале):
- `portalUrl`, `portalLoginUrl`, `portalApkCategoryId`, `portalManifestFileId`, `portalFileDownloadUrl()`, `portalCredentials` (с паролем), `updateManifestUrl` (поле и в конструкторе).
- Проверка обновления больше не настраивается через URL — endpoint 1С фиксирован: `inventoryPath('update')`.
Все остальные поля (`baseUrl`, `remoteUrl`, таймауты, `inventoryPath`, `keyForUrl` и т.д.) не трогать.

### 2. `lib/core/update/domain/version_manifest.dart`
Переписать под новый контракт:
- Поля: `int versionCode`, `String versionName`, `String apkUrl`, `String releaseNotes`, `String sha256`, `int urlExpiresInSec`, `bool required`.
- Удалить `apkFileId` и `resolvedApkUrl`.
- Безопасный парсинг: некорректный `versionCode`→0; отсутствующие строки→`''`; `required` по умолчанию `false`; `urlExpiresInSec`→0; битый JSON не валит приложение (восстановление через defaults).
- `isNewerThan(int current)` — сравнение строго по `versionCode`.
- `get isValid` → `sha256.isNotEmpty && apkUrl.isNotEmpty` (для ранней отсечки некорректного манифеста).

### 3. `lib/core/network/api_error.dart`
Добавить `IntegrityError extends ApiError` → userMessage «Целостность APK нарушена (SHA-256 не совпал)».

### 4. `lib/core/update/data/update_repository.dart`
Полная переработка:
- Конструктор: `UpdateRepository({required DioClient client, Dio? downloadDio})`.
  - `client` — авторизованный DioClient 1С для запроса манифеста.
  - `downloadDio` — отдельный чистый `Dio()` для скачивания APK по подписанной ссылке (без Basic Auth, без cookies, без X-Update-Token).
- Удалить весь WordPress-код: `_wpLogin`, `_ensureCookie`, `_portalHeaders`, `_cookie`, `credentials`, `login`.
- `checkForUpdate(String endpointUrl)` → `Future<Result<VersionManifest>>` через `client.getJson(endpointUrl)`, безопасный парсинг.
- `downloadApk(String apkUrl, {required String sha256, Directory? targetDir, DownloadProgress? onProgress})` → `Future<Result<File>>`:
  - если `sha256` пустой → `Failure(ParseError(...))` (не начинаем качать);
  - качает через чистый `downloadDio` без авторизационных заголовков;
  - после скачивания считает SHA-256 файла и сравнивает без учёта регистра;
  - при несовпадении — удаляет файл, возвращает `Failure(IntegrityError())`;
  - при ошибке сети/таймаута/истёкшей ссылке — `Failure(ApiError.fromDio(...))`, `deleteOnError: true`.

### 5. `lib/core/update/application/update_controller.dart`
- Удалить зависимость от `_config.updateManifestUrl` (фича всегда включена после авторизации).
- Guard от повторных параллельных проверок: если `state is UpdateChecking || state is UpdateAvailable || state is UpdateDownloading || state is UpdateInstalling` → не запускать новую `checkAndPrompt()`.
- `checkAndPrompt()` дёргает `_repo.checkForUpdate(config.inventoryPath('update'))`; флаг `_dialogShown` НЕ хранить в контроллере (это ответственность UI, чтобы не делать контроллер UI-aware).
- `downloadAndInstall()` использует `manifest.apkUrl` и передаёт `manifest.sha256` в `downloadApk`; если `!manifest.isValid` → сразу `UpdateError('Манифест обновления некорректен')` без скачивания.
- `skip()` доступен, только если `manifest.required == false` (UI этим пользуется).
- Провайдер `updateControllerProvider` построен через сессию: если `session == null` → кидать `StateError` (провайдер читается только на DocsListScreen после входа). Передаёт `UpdateRepository(client: DioClient(config, BasicCredentials(session.login, session.password)))`.

### 6. `lib/core/update/presentation/update_dialog.dart`
