## Функция снятия факта сканирования позиции

**UX:** долгое нажатие (long-press) на карточке отсканированной позиции (где `qtyActual > 0`) открывает диалог с тремя кнопками в вертикальном стеке (по паттерну `_showNotFound`):
- **«Убрать единицу (−1)»** — заполненная кнопка сверху (безопасное действие), декремент факта;
- **«Сбросить факт (=0)»** — outline снизу, деструктивная (красная), обнуляет факт;
- **«Отмена»** — outline, закрывает диалог.

Long-press включается только для найденных позиций (`isFound`), чтобы не предлагать снять то, чего нет.

### Изменения по файлам

**1. `lib/features/inventory/application/scan_controller.dart`**
- Добавить приватный `_setActual(DocTableRow row, int value)`: находит строку по `lineNumber`, делает `copyWith(qtyActual: value)` (с защитой от отрицательных), персистит через `_db.upsertScanProgress(...)`, вызывает `notifyListeners()`. Переиспользует уже существующий паттерн персиста из `onScanned`/`applyChoice`.
- Добавить публичные:
  - `Future<void> decrementActual(DocTableRow row)` → `max(0, qtyActual - 1)`;
  - `Future<void> resetActual(DocTableRow row)` → `0`.
- Поле `action` намеренно не трогается (серверное поле расхождения), как и в `_incrementActual`.

**2. `lib/features/inventory/presentation/row_card.dart`**
- Добавить опциональный параметр `onLongPress` (`VoidCallback?`).
- Обернуть `Card` в `InkWell`/`GestureDetector` с `onLongPress` (только когда колбэк передан). Иначе поведение карточки не меняется.

**3. `lib/features/inventory/presentation/inventory_screen.dart`**
- В `_Body` при построении `RowCard` передавать `onLongPress`: если `rows[i].isFound`, колбэк вызывает новый метод `_showUnscanDialog(rows[i])`, иначе `null`.
- Добавить метод `_showUnscanDialog(DocTableRow row)`:
  - кастомный `AlertDialog` по образцу `_showNotFound` (вертикальный стек кнопок, единые высоты/радиусы);
  - title — номенклатура, content — «Факт: N» (через `AppStrings.qtyActualOf`);
  - кнопки: декремент (`_scan?.decrementActual(row)`), сброс (`_scan?.resetActual(row)`, деструктивный цвет `scheme.error`), отмена.
- Диалог закрывается перед вызовом действия (как в `ConfirmDialog.show`).

**4. `lib/l10n/app_strings.dart`**
- Добавить строки: `unscanDecrement` = 'Убрать единицу (−1)', `unscanReset` = 'Сбросить факт (=0)' (переиспользуем существующие `cancel` и `qtyActualOf`).

### Поведение / edge-кейсы
- Сброс в 0 → `isFound` становится `false`, карточка теряет зелёную подсветку и при включённой сортировке «сначала неотсканированные» уезжает вниз списка (ожидаемо).
- Прогресс `scannedCount`/`isFullyScanned`/`hasDiscrepancies` пересчитываются автоматически, т.к. зависят от `qtyActual`.
- После сброса запись прогресса остаётся в БД с `qtyActual = 0` (безвредно, `hydrateFromDb` корректно поставит 0) — это минимизирует изменения в БД и точки отказа.
- Тактильный/звуковой feedback не добавляется (не относится к результату сканирования).

### Что НЕ меняется
- Логика сканирования, матчинга, отправки (`commit`), добавления номенклатуры.
- Структура БД (новые методы/таблицы не нужны — используем существующий `upsertScanProgress`).
