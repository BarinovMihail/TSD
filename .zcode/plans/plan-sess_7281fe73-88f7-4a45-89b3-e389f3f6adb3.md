## Контекст

Архитектура изучена: `DioClient` (Basic Auth, failover), `InventoryRepository` (возвращает `Result<T>`), `DocTableRow`/`parseDocTable`, `ScanController` (ChangeNotifier), Riverpod (`inventoryScreenControllerProvider.family`), drift (`AppDatabase`, схема v2 — без изменений), `RowCard`, экран `InventoryScreen` + `_Body`. Иконки `assets/icons/barcode_available.png` и `barcode_missing.png` уже лежат в репо.

Решение по мёртвому коду (согласовано): **удалить** старый per-scan путь (`getBarcodeInfo`, `BarcodeInfo`, `addNewLine`, `ScanController.addMissingLine`, `_showNotFound`/`_addMissingLine`, `matchByNomenclatureCharacteristic`, устаревшие `ScanOutcome`). Схема БД не меняется — штрихкоды приходят с сервера при каждой загрузке документа, локально хранится только прогресс (факт/действие).

## Реализация

### Домен
**`domain/doc_table_row.dart`**
- Добавить поле `final List<String> barcodes;` (в конструкторе приводить к `List.unmodifiable`).
- Геттер `bool get hasBarcodes => barcodes.isNotEmpty;`.
- `copyWith`: сохранить `barcodes` (+ опциональный `List<String>? barcodes` для обновления после reload).

**`domain/doc_table_parser.dart`**
- Читать «Штрихкоды»: если `List` → `.toString().trim()`, отбросить пустые, **как строки** (без `int`, без потери нулей); иначе `const []`. null/отсутствует → `[]`.

**`domain/barcode_matcher.dart`** — переписать
- Оставить `MatchResult` (isUnique/isNone/isAmbiguous).
- Новый метод `MatchResult matchByBarcode(String code, List<DocTableRow> rows)`: `trim` отсканированного; пусто → `isNone`; сравнение `r.barcodes.any((b) => b.trim() == c)` (строковое, с учётом ведущих нулей); несколько строк с тем же кодом → `isAmbiguous`.
- Удалить `matchByNomenclatureCharacteristic`, `normalize`, `normalizeCode`.

**`domain/barcode_info.dart`** — удалить файл.

### Данные
**`data/inventory_repository.dart`**
- Удалить `getBarcodeInfo`, `addNewLine`.
- Добавить `Future<Result<List<String>>> getCharacteristics(String nomenclature)` → `GET hs/inventory/invent/${Uri.encodeComponent(...)}`; ответ — JSON-массив строк → `trim`, отбросить пустые; не-List/ошибка → `ParseError`; `DioException` → `ApiError.fromDio`.
- Добавить `Future<Result<void>> addBarcode(String nomenclature, String characteristic)` → `POST hs/inventory/newBarcode`, тело `{'Номенклатура': nom, 'Характеристика': char}` (пустая строка для «Без характеристики»).
- Оставить `getTable`, `postDocResult`, `getCurrentUserFio`.

### Приложение
**`application/scan_controller.dart`** — переписать
- `ScanOutcome`: оставить `Found(row)`, `Ambiguous(candidates)`, `NotFoundInDocument(code)` (= «штрихкод не найден в массивах строк»); добавить `ScanIgnored()` (пустой после trim); удалить `BarcodeNotRegistered`, `LookupError`.
- `onScanned(code)`: `trim`; пусто → `ScanIgnored`; иначе `_matcher.matchByBarcode(code, rows)` → unique: +1 факт + persist + `Found`; ambiguous: `Ambiguous`; none: `feedback.error()` + `NotFoundInDocument`. Локально, без сети.

- `_incrementActual/_setActual/decrement/reset/hydrateFromDb/replaceRows/commit/applyChoice` — без изменений.

**`application/inventory_screen_controller.dart`**

- `reload()` — без изменений (повторно парсит «Штрихкоды» через `replaceRows`, прогресс не теряется).
